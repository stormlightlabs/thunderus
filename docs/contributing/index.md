# Contributing

We welcome contributions! Do your best to make sure your contributions emphasize
reviewability, safety, and clear diffs.

## Build and Test

```sh
cargo build
cargo test
```

For faster feedback:

```sh
cargo check
```

Formatting and linting:

```sh
cargo fmt
cargo clippy
```

## Project Structure

For crate responsibilities and boundaries, see [Development Architecture](/development/architecture).

If you are new to the codebase, start with the [Development guide](/development/).
